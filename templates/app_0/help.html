How administration works

  1. Create a Site (global container for your organizations).
  2. Create Organizations under a Site.
  3. Roles define capabilities, default ones are Site Admin, Org Admin, Member.
  4. Memberships link a User to a Site and optionally an Organization with a Role.

Users can have multiple memberships across different sites/organizations/roles.

Tips
  • Use Memberships to assign Site Admin at site level (leave Organization blank).
  • Assign Org Admin by choosing a specific Organization on the Membership.
  • Soft-delete is supported; use the active flag to disable entries.
  • UI quick start: go to Site Admin, click New Site, then Assign admin, then add Organizations and assign Org Admins via the modals.
  • CLI: use management command bootstrap_site with a JSON file to load sites, orgs and memberships for testing or production seeding.
